    Current signature (unchanged):
    ```kotlin

    fun CurrentTasksView(
        tasks: List<Task>,
        onRestart: () -> Unit,
        onToggleComplete: (Task) -> Unit
    )
    ```

    Inside `CurrentTasksView`, before the `Scaffold`, derive the all-complete state:
    ```kotlin
    val priorityTasks = tasks.filter { it.isPriority }
    val allComplete = priorityTasks.isNotEmpty() && priorityTasks.all { it.status == "COMPLETED" }

    // One-shot: only show confetti once when the transition to allComplete happens.
    // Reset when tasks change (new session loaded).
    var confettiShown by remember(tasks.map { it.id }.toSet()) { mutableStateOf(false) }
    val showConfetti = allComplete && !confettiShown
    ```

    Wrap the existing `Scaffold(...)` block in a `Box(modifier = Modifier.fillMaxSize())`. After the Scaffold, add the KonfettiView as a sibling in the Box so it overlays full-screen:
    ```kotlin
    Box(modifier = Modifier.fillMaxSize()) {
        Scaffold(
            floatingActionButton = { ... }
        ) { padding ->
            // existing LazyColumn content — unchanged
        }

        if (showConfetti) {
            val parties = listOf(
                Party(
                    emitter = Emitter(duration = 3, TimeUnit.SECONDS).perSecond(100),
                    position = Position.Relative(0.5, 0.0)
                )
            )
            KonfettiView(
                modifier = Modifier.fillMaxSize(),
                parties = parties,
                updateListener = object : OnParticleSystemUpdateListener {
                    override fun onParticleSystemEnded(system: nl.dionsegijn.konfetti.core.models.PartySystem, activeSystems: Int) {
                        if (activeSystems == 0) confettiShown = true
                    }
                }
            )
        }
    }
    ```

    IMPORTANT details:
    - Remove the existing `val priorityTasks = tasks.filter { it.isPriority }` line from inside the Scaffold's LazyColumn section — it is now computed before the Scaffold. Update any reference to `priorityTasks` inside the LazyColumn to use the same variable (they are in scope).
    - `remember(tasks.map { it.id }.toSet())` resets `confettiShown` to false when a new session is loaded (task IDs change), so confetti fires again on the next completion.
    - The `KonfettiView` is rendered on top of the Scaffold but does not consume input — touch events pass through to the task list. Do NOT add `.clickable {}` or `pointerInput` to the KonfettiView modifier.
    - `OnParticleSystemUpdateListener.onParticleSystemEnded` is called when the emitter finishes. Setting `confettiShown = true` there prevents replays.
    - Keep the `val otherTasks` and all existing LazyColumn structure inside the Scaffold unchanged.

    If `OnParticleSystemUpdateListener` is abstract and requires more overrides in this version of konfetti-compose, implement all required methods with empty bodies. Check the konfetti-compose 2.0.5 API — `onParticleSystemEnded` may have a different signature; adapt accordingly.
