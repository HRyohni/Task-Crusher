  <name>Task 1: Fix hasSeenBackTapGuide initial value (BUG-02)</name>
  <files>app/src/main/java/hr/fipu/organizationtool/ui/TaskViewModel.kt</files>
  <action>
    Line 31 of TaskViewModel.kt reads:

    ```kotlin
    val hasSeenBackTapGuide: StateFlow<Boolean> = onboardingRepository.hasSeenBackTapGuide
        .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5000), true)
    ```

    The initial value `true` is used while the DataStore Flow has not yet emitted. On a fresh
    install, the DataStore has no stored value and emits `false`, but before that emission arrives,
    `stateIn` returns `true` — causing `ZenStackApp.kt` ConfirmationStep to call `onFinished()`
    instead of navigating to `currentStep = 4` (BackTapOnboarding).

    Change the initial value from `true` to `false`:

    ```kotlin
    val hasSeenBackTapGuide: StateFlow<Boolean> = onboardingRepository.hasSeenBackTapGuide
        .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5000), false)
    ```

    No other changes to this file in this task (the saveSession reset was applied in plan 07-01).
  </action>
  <verify>

    Grep confirms `stateIn(viewModelScope, SharingStarted.WhileSubscribed(5000), false)` appears
    for hasSeenBackTapGuide.
    `./gradlew :app:assembleDebug` succeeds.

  <name>Task 2: Add Accessibility Settings deep-link button (UX-02)</name>
  <files>app/src/main/java/hr/fipu/organizationtool/ui/components/BackTapOnboarding.kt</files>
  <action>
    BackTapOnboarding currently shows manual step-by-step instructions (Steps 1-4) and a single
    "Got it" button. Add a primary button above "Got it" that opens Android Accessibility Settings
    directly, so the user does not need to navigate manually.

    Changes to BackTapOnboarding.kt:

    1. Add imports at the top of the file:
       ```kotlin
       import android.content.Intent
       import android.provider.Settings
       import androidx.compose.ui.platform.LocalContext
       ```

    2. Inside the `BackTapOnboarding` composable function body, obtain context:
       ```kotlin
       val context = LocalContext.current
       ```

    3. Replace the existing `Spacer(modifier = Modifier.height(48.dp))` before the "Got it" button
       with a smaller spacer, and insert an "Open Accessibility Settings" OutlinedButton before it:

       ```kotlin
       Spacer(modifier = Modifier.height(32.dp))

       Button(
           onClick = {
               context.startActivity(
                   Intent(Settings.ACTION_ACCESSIBILITY_SETTINGS).apply {
                       flags = Intent.FLAG_ACTIVITY_NEW_TASK
                   }
               )
           },
           modifier = Modifier.fillMaxWidth(),
           shape = RoundedCornerShape(12.dp)
       ) {
           Text("Open Accessibility Settings", style = MaterialTheme.typography.titleMedium)
       }

       Spacer(modifier = Modifier.height(12.dp))

       OutlinedButton(
           onClick = onGotIt,
           modifier = Modifier.fillMaxWidth(),
           shape = RoundedCornerShape(12.dp)
       ) {
           Text("Got it", style = MaterialTheme.typography.titleMedium)
       }
       ```

       The existing primary `Button` for "Got it" becomes an `OutlinedButton` (secondary action).
       The new primary `Button` opens Accessibility Settings. This visual hierarchy communicates
       that the recommended action is to open settings, while "Got it" is the dismissal path.

    The instruction steps (InstructionItems 1-4) remain unchanged — they still explain what to do
    once the user is in settings.
  </action>
  <verify>

    3. The "Got it" action is an OutlinedButton (not a primary Button).
    4. `./gradlew :app:assembleDebug` succeeds with no Compose or import errors.
  </verify>
  <done>
